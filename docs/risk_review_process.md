# Risk Review Process

The repo models a lightweight program risk board.

Each record captures:

- a unique risk ID
- affected workstreams
- review gate ownership
- initial scoring
- residual scoring
- mitigation actions

The expected review flow is:

1. engineering lead raises the risk
2. board assigns ownership and gate
3. mitigation plan is tracked
4. residual exposure is reviewed before closure

This makes the repository useful both as a portfolio artifact and as a discussion tool in interviews about risk burndown, governance, and review discipline.
